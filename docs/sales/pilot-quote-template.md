## 3. 项目范围

### 3.1 包含项（试点交付范围）

- 现场网络与路由诊断，输出可重复运行的 `test-plc-network.ps1` 报告。
- S7 协议握手验证（`connect-plc.ps1`），记录现场可用的 `connection-type / rack / slot`。
- 设备信息读取（`--device-info`），记录块数量与不支持接口列表。
- 配置校验（`--validate-config`）覆盖客户实际项目 JSON。
- 单次只读读取（`--read-once`）覆盖现场关键点位。
- 有限轮询读取（`--cycles N`），输出 JSONL 结构化运行日志。
- 旧点位表导入（如有）：从客户旧 XML / 旧上位机配置生成新项目 JSON，并完成只读冒烟。
- 交付物打包：项目 JSON、点位表、运行日志样例、诊断脚本、启动说明、现场验收记录。

### 3.2 协议范围

- **主线**：Siemens S7（S7-200 SMART 已验证；S7-1200/1500 按现场调整 rack/slot/connection-type）。
- **第二阶段（按客户要求另议）**：Modbus TCP 真实设备验证。当前本地 loopback 已通过，但真实 Modbus 设备需在现场单独确认 UnitId、地址起始、字节序，**不作为试点默认承诺项**。

### 3.3 点位范围

- 第一阶段：30 个关键只读点位（参考 `legacy_imported_smoke.project.json` 的实测规模）。
- 第二阶段：按客户认可扩展到完整点位表（设备数 × 协议数 × 点位数，按本表第 4 节里程碑追加报价）。

### 3.4 运行范围

- 试点阶段：单次读取、有限轮询、结构化 JSONL 日志。
- **不包含**：连续 24 小时运行、Windows Service 部署、SQLite/数据库持久化、断线自动重连、上层 UI、客户系统对接。这些归属正式项目报价（见 §11）。

收款节奏建议：

- M1 启动款不少于 **30%**（覆盖现场动员、行程、设备准备的不可逆成本）。
- M2 演示款 **30%–40%**（与现场连接成功强相关，是客户最容易感知价值的节点）。
- M3 验收尾款 **30%**（与点位完整性、交付物结构化相关，不要 < 20%，否则验收阶段无议价空间）。

如有差旅、住宿、专项设备购置（如临时交换机、长网线、串口转换器），按 §7 单独列示，**不并入项目款**。

## 5. 不包含项（明确范围以避免蔓延）

以下事项**不在本次试点报价范围内**，如客户要求，需另行报价：

- 真实 PLC 写入演示与验证（需客户提供安全测试点位，作为独立增量包）。
- Windows Service / 开机自启 / 守护进程部署。
- SQLite 或其他数据库的持久化存储。
- 自动断线重连、降频重试、多设备状态机。
- 任何上位机界面（Qt / WPF / Web / Avalonia）。
- 与客户 MES / SCADA / ERP / 报表系统的对接、数据上送、API 接口。
- 客户网络改造、PLC 程序修改、PLC 通信参数调整（PUT/GET 启用等）。
- 7×24 小时驻场或电话支持。
- 任何 SaaS、订阅、平台、云端服务。
- 培训客户员工独立维护（如需，按运维包另议）。

## 6. 验收口径（按里程碑）

### M1 启动款验收口径

- 双方确认本报价书签字。
- 客户提供：设备清单、PLC IP 列表、可进场日期、现场 IT 联系人。
- 客户书面确认前置条件（见 §9）。

### M2 现场演示款验收口径

- 现场可重复执行 `tools\test-plc-network.ps1 -Ip <PLC IP>`，输出 `TcpTestSucceeded: True` 且 `InterfaceAlias` 为物理网卡。
- 现场可重复执行 `tools\connect-plc.ps1`，输出 `SUCCESS: Snap7 connected with connectionType=... rack=... slot=...`。
- 现场可重复执行 `tools\run-s7-demo.ps1 --adapter snap7 --device-info`，输出至少 `BlockCounts` 一行。
- 现场可重复执行 `tools\run-s7-demo.ps1 --adapter snap7 --read-once`，默认探测点位全部 `GOOD`（个别 PLC 探测点不可读时以备注形式说明，不计入失败）。
- 现场可重复执行 `tools\run-s7-demo.ps1 --adapter snap7 --cycles 2 --interval 1 --run-log <path>`，生成 JSONL 文件且每行为合法 JSON。
- 交付现场诊断快照（截图 + 命令输出 + JSONL 样本）。

### M3 验收尾款验收口径

- 客户业务点位表已通过 `import-legacy-config.ps1` 导入或人工配置为新项目 JSON。
- 至少 **30 个**关键业务点位执行 `--cycles 1 --fail-on-bad-quality` 退出码为 `0`（即全部 `GOOD`）。
- JSONL 运行日志已交付，包含 `type=snapshot, timestampUtc, deviceId, good, bad, values[]` 字段。
- 交付项目 JSON、点位表、启动脚本、运行日志样例、现场验收记录（按 `docs\sales\pilot-acceptance-checklist.md` 逐条签字）。
- 客户决策人签字确认验收通过。
